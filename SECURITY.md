# Security Policy

## Reporting Security Vulnerabilities

We take security seriously. If you discover a vulnerability in Sophis, please report it responsibly.

### Reporting Channels
1. **Email**: security@sophis.org (PGP-encrypted preferred)
2. **GitHub**: Use the private vulnerability reporting feature

### What to Report
- Smart contract vulnerabilities
- Consensus mechanism issues
- Network attack vectors
- Cryptographic weaknesses
- Any security-relevant bugs

### Response Timeline
| Severity | Acknowledgment | Initial Assessment | Fix |
|----------|---------------|-------------------|-----|
| Critical | 4 hours | 24 hours | 48 hours |
| High | 24 hours | 72 hours | 7 days |
| Medium | 48 hours | 1 week | 30 days |
| Low | 1 week | 2 weeks | Next release |

### Disclosure Policy
1. Report vulnerability privately
2. Maintainainer confirms and triages
3. Fix developed and tested
4. Coordinated disclosure after fix is released
5. Reporter credited (if desired)

### Out of Scope
- Social engineering
- DoS requiring massive resources
- Issues in third-party dependencies (report upstream)

Thank you for helping secure Sophis.
